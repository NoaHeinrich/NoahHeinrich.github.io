Hashing Things Out: Array of Sunshine

9/30/15

Arrays and hashes can be a little conceptually difficult for the uninitiated. Heck, they still confuse me a bit, but my research here can show you what the difference is, and the reasons to use one over the other.

An array is a collection of items that Ruby saves for you, and lets you access later. These items can be integers, floats, or strings, it treats them both the same. It looks very simple:

      new_array = [1, 2.0, "Buckle my shoe"]

You can tell Ruby to go through the entire list, or pick out just one by calling on its place in line. In our example, 1 has the place of 0, because all Ruby lists start at 0. Use an array to store simple information. Arrays are very useful for any sort of iterative program that you want to make, using the .each function. 

      new_array.each {|x| puts x}

This line of code will put every item within new_array onto the screen. It's simple, but the principle of it can be used for more complicated code.

Hashes, on the other hand, are a bit more complicated. Like arrays, they host a number of items, and those items can be any number or string. However, instead of just storing them in a line, a hash asks you to assign some sort of descriptor to everything you want to store. In essence, a hash is like two different arrays, connected at the hip. 

      ident = { "First Name" => "Noah",
        "Last Name" => "Heinrich",
        "Age" => 25,
        "Hometown" => "Detroit"
      }

You can use this to store data rather nicely. It's like your own personal little index that you can access at your leisure. Each item is linked to the second item. The first items act as the key, if you will, to the second item. I could write a little program to display the information. 

      ident.each do |x, y|
        puts "Your #{x} is #{y}."
      end

The output of this code would look like this: 

      Your First Name is Noah.
      Your Last Name is Heinrich.
      Your Age is 25.
      Your Hometown is Detroit.

Impressive, right? Alright, I'll admit that I'm still a little shaky on what practical things you can do with arrays and hashes. I'm still at the stage where I'm making cute little games and calculators with Ruby. However, I can see the glimmerings of what can be. You can use hashes as the basis for databases, and use arrays to analyze the contents of a written message, or search for specific numbers or words. I don't know how to do these things yet, but you, like me, now know the difference between an array and a hash, and can use that knowledge to further your education!
